Areas de GIT

Working Directory

El directorio de trabajo corresponde a la carpeta donde inicializamos el repositorio de git. Muy pocas veces vamos a estar utilizando la linea de comandos, ya que normalmente corresponde al area en la que se crean, editan y eliminar los archivos. Algunos de los comandos que podemos llegar a utilizar son los siguientes:

  • dir || ls: muestra el contenido de la carpeta
  • mkdir <name>: crea una carpeta con el nombre especificado
  • cd <path>: abre la carpeta seleccionada
  • git init: inicializa un repositorio de git
  • rm -rf .git elimina el repositorio de git

Staging Area (INDEX)

El area de control de cambios se utiliza para identificar los archivos a los que vamos a hacerle seguimiento, aqui realizamos las capturas de codigo, que luego deberan ser confirmadas para formar parte del historial de confirmaciones. Entre los camandos a utilizar se encuentran

  • git status: muestra el estado de los archivos con respecto al INDEX
  • git add <file>: realiza la captura (snapshot) de los archivos seleccionados
  • git diff: compara el Working Directory con el Staging Area
  • git restore --staged <file> elimina la ultima captura realizada del archivo seleccionado
  • git rm --cached <file> elimina del repositorio las captura del archivo seleccionado

Repository (Local/Remote)

El repositorio corresponde al almacen de confirmaciones donde se ubican las capturas de codigo confirmadas, en archivos BLOB, identificados con un numero de HASH generado a partir del nombre del autor, correo, cambios realizados, fecha y hora de creacion. Los comandos que vamos a estar utilizando para administrar el repositorio son los siguientes:

  • git log: muestra el historial de confirmaciones
  • git blame <file>: muestra los autores de cada linea de codigo del archivo seleccionado
  • git remote add <alias> <url>: agrega una direccion de un repositorio remoto
  • git fetch <remote> captura el historial de confirmaciones del remoto
  • git pull <remote> <branch> integra los cambios del remoto al working directory
  • git push <remote> <branch> envia el historial de confirmaciones al repositorio remoto
